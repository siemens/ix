Dos and dont's

Vocabulary​

We use vocabulary that supports a conversational, natural language, i.e. human not robotic.

American English
User-specific terminology
Repetition in the same product: create + create + create
Short words: help, support
Buttons with 1 or 2 words
Gender-neutral words: you, your, yours, we, us, our/s
Anomaly, event, event type, mobile device
Predictable words: save, delete, add, renew, train, model, use
Context-based: 'please' and ‘sorry’ only for inconvenient/unplanned events
Other Englishes unless explicitly requested
Buzz words, jargon, jokes: oops! Uh oh, we lost your data!
Synonyms in the same product: make + build + create
Long words: assistance
Wordy buttons with additional adjectives or adverbs: Click here to quickly save
Gender-specific words: he, she, his, hers, him, salesman
Problem, issue, smartphone
Vague words: should, might, may, would, could
Using 'please' to request basic user actions: Please enter username

Grammar​

We use simple grammar constructions and clauses to increase readability.

Active voice: Configuration file opens.
Present simple tense: click, upload, browse
Simple verb forms to confirm actions: File loaded. File will load.
Positive contractions: you'll, we’ve
Siemens as singular: Siemens has/is
Single clauses: Browse log file.
Passive voice: Configuration file is opened.
Complex tenses: clicking, being clicked, was clicking, had been clicked
Complex verb forms: File is going to be loaded. File has been loaded.
Negative contractions: can’t, won't
Siemenss / Siemens' / Siemens's are
Multiple and complex clauses: Browse to the file in which the log has been inputted.

Wordiness​

We capitalize the first letter (sentence case) and all special names (proper nouns) to simplify the UX writing process.

Capitalize the first word and any proper noun: Contact your Siemens support.
Capital letters for logos, products, roles, titles: MindSphere, eBR, SIMATIC
OK
n/a or N/A
Capital letter after colons: Log in
Capital letters after each bullet point in lists
Capitalize functions when referred to in a text: Allocate users in User management
Capitalize everything: Contact Your Siemens Support.
No changing the casing of a product: Mindsphere, Ebr, Simatic
Ok or Okay
N/a, n.a., N.A.
Small letter after colons: log in
Small letters after each bullet point in lists
Inconsistent upper and lower casing for functions: Allocate users in user management
